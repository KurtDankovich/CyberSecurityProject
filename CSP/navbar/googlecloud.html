Hosting Static Website on Google Cloud

Obtain Domain Name

The first step in order to host a static webpage on Google Cloud is to acquire a domain name from a domain name hosting organization such as Google Domains, Bluehost, GoDaddy, etc. I decided to purchase the domain dankcsp.org through Bluehost.

Verify Domain Name

The next step is to verify the domain through Google Search Console. This process involves typing the acquired domain name into the section that says "Domain". Google will then bring up a popup that will provide a CNAME or TXT record. It is important to copy either the CNAME or TXT record.

Once the CNAME or TXT record is copied. Head over to the domain hosting organization site in which the domain was purchased (Bluehost in my case). From there, click on the domain acquired and navigate to where there is CNAME or TXT record entry. Paste the record into the correct entry whether that be CNAME or TXT. After some time, there should be a Ownership Auto Verified message that says domain verification was successful. Figure 2.3 shows the message that should pop up if the verification was successful.

Create a Bucket

To host a static website on the Google Cloud, a bucket must be created to hold the code that makes up the website. To create a bucket the first thing while at the Google Cloud Console is to click on the console search bar and type "Buckets". Click the option that says Buckets. Once at the buckets page, click "Create".

Name the bucket exactly whatever domain was verified from earlier(for me it was dankcsp.org). There is an option where to store the data however I choose the default option which was multiregion.

Upload Code

The newly created bucket will appear at the main bucket page. From here, click on the new bucket. This will give a space where code can be uploaded and stored within the bucket. Once uploaded the server will take minutes to update and the domain name should be accessible.
