About Project

Introduction

The technological world has been growing for the last few decades at a very fast pace. With the growth of new technologies, more sophisticated threats started to become developed which could be hazardous to the systems that are used by the average user or even damage the security of a massive organization. One method of securing a network from unwanted threats is to create a honeypot network.

What is a Honeypot?

Norton.com describes a honeypot as a “security tool that can help computer systems be defined against cyber attacks in unique ways.”(NortonOnline) A honeypot is a decoy network that contains systems that are posing as real user systems. Attackers will target these honeypots as they believe they are real vulnerable systems that they can steal information that could be useful to the attack. A honeypot is used in order to gain vital information on an attacker’s behavior.

Benefits of a Honeypot

A honeypot has several benefits that include giving a user or organization the visibility to see what exactly a firewall is missing. IT professionals will be able to use the honeypot to create new rules that can block any activity from the IP addresses that are trying to attack the network. The honeypot also gives a user or organization the methods that attackers are using to attack their network. Honeypots also waste an attacker's time and resources which will take away time they spend attacking the real network.

T-Pot Honeypot

The honeypot I am choosing to deploy is T-Pot that was developed by Telekom Security. T-Pot is an all in one honeypot that can be deployed on a cloud service using a virtual machine. T-Pot requires at least 8GB of RAM while needing 128GB of storage. There are two methods of deploying this honeypot. One method is to use a dedicated physical machine to run the honeypot. The second method is to run a virtual machine on a cloud service and deploy T-Pot to the virtual machine. This method requires a cloud service and internet access. I decided to use the cloud method as I do not have any spare physical machine. Also, cloud services use a pay as you go method for how much resources are used.

Cloud Service

T-Pot requires a system to be run on. The way to implement this is to use a cloud service. The cloud is a series of computer system resources that can be used by an end user. For example, Google runs the Google Cloud which is a bunch of physical systems that are in place for a user to use with minimal physical user hardware. The user does not have to perform any system maintenance as the cloud provider deals with the systems on their end. Some benefits of using a cloud service are cost to operate, more security as the cloud provider provides necessary security, and loss prevention. In my case, using a cloud service will prevent my real system from being attacked and if anything goes wrong I can keep my physical systems safe as I can delete the virtual machine instance that T-Pot resides on.

Why Google Cloud

Google Cloud is a cloud service that is maintained by Google. They offer $300 credit over 90 days for new users. They also have no autocharge after the free trial ends. Google Cloud also gives $100 credit if an organization account is linked to the email used. In this case, I used my student email which qualified me for an extra $100. This $400 credit will be enough to host a virtual machine that has 8GB RAM and 128GB of storage. Google Cloud has all the necessary tools in order to get T-Pot up and running. Google Cloud also secures my personal network as they do not give virtual machines IP addresses on my home network. They use VPNs in order to give a different IP address.

Virtual Machine

A virtual machine is a computing resource that uses software instead of physical machines to run its own instance with its own operating system. A virtual machine allows for a system to run like a whole new system without changing the original system. A virtual machine can be run in software such as VMware or can be run using a cloud service. On Google Cloud, I will be making a virtual machine that runs Debian 11. Figure 1.2 shows the virtual machine set up. The virtual machine has 8GB ram, 128GB storage, is running Debian 11 and will cost $61.72 a month.

Summary

This project will consist of deploying T-Pot honeypot to a Debian based virtual machine that is being hosted on Google Cloud. This project will need a virtual machine that has 8GB RAM, 128GB of storage, and Debian 11. T-Pot installation and all necessary files can be found at T-POT. The purpose of this project is to see what, where and who is attacking the honeypot and to analyze the data after 2 months of leaving T-Pot on.
